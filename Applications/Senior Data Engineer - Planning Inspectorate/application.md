Response B — Technical Example (250 words)
```
At Accelercomm, engineering teams needed reproducible 5G NR physical-layer test vectors to support continuous regression testing and demonstrate compliance with 3GPP standards. We needed reproducible test results and clear traceability to demonstrate the product features we were presenting to the customer.

I designed and implemented a Python workflow for MATLAB-generated test vectors. The CI/CD system autogenerated vectors using the MATLAB CLI, orchestrated entirely through a Python toolchain. I defined JSON schemas in Python to validate and structure outputs, ensuring consistent formats before data entered the pipeline. Generated vectors could then be used for future test runs and regression testing. The workflow integrated with Jenkins so that vector generation ran automatically on each build. Outputs were archived in Artifactory, providing version-controlled, cloud-accessible storage that teams could rely on for repeatability.

Downstream, pytest and Google Test frameworks consumed the archived vectors to validate 3GPP physical channels across hardware accelerator and software subsystems. I applied version control and automated testing throughout, and worked with stakeholders to align data requirements with compliance needs.

This delivered a reliable, governed data pipeline: reproducible test data on every CI run, reduced manual effort, improved traceability, and stronger confidence in compliance evidence across the organisation.
```
